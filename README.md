# 📊 Data Warehouse & Analytics Project

## 📌 Overview

In this project, I built a complete data warehousing and analytics solution, starting from raw data ingestion to generating meaningful business insights. The goal was to simulate a real-world scenario where data from multiple sources is consolidated, cleaned, and transformed into a format suitable for analysis and reporting.

This project reflects practical data engineering and data analysis workflows commonly used in industry.

---

## ⚙️ Project Requirements

### 🔹 Data Warehousing (Data Engineering)

#### Objective

The objective was to design and build a data warehouse using SQL Server by combining data from multiple sources. This enables structured reporting and supports better decision-making.

#### Key Steps

* Imported data from different systems (ERP and CRM) provided as CSV files
* Cleaned and handled data quality issues before analysis
* Integrated multiple datasets into a unified data model
* Designed the data model in a way that is optimized for analytical queries
* Focused only on the latest dataset (no historical tracking required)
* Documented the structure clearly for both technical and business users

---

### 📈 Analytics & Reporting (Data Analysis)

#### Objective

After building the data warehouse, I used SQL to analyze the data and extract useful insights.

#### Analysis Focus Areas

* Customer behavior
* Product performance
* Sales trends

These analyses help in understanding key business patterns and support strategic decision-making.
